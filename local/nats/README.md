# NATS Event Bus Deployment

NATS deployment configuration for the DSX Event Bus evaluation.

## Overview

NATS is deployed across Kubernetes layers (CSC, CPC) with:
- MQTT support for MQTT 3.1.1 protocol
- JetStream for message persistence
- Leaf nodes for federation between layers
- Network and gateway-level isolation (separate clusters per layer)

## Architecture

Each layer runs a 3-node NATS cluster with MQTT support, JetStream for persistence, and leaf nodes connecting CPC clusters to CSC for federation.

## Deployment

### Prerequisites

- Kind clusters created (CSC, CPC-1, CPC-2)
- Helm 3.12+
- kubectl configured with cluster contexts

### Deploy to All Layers

```bash
# From the repository root
make -C local deploy-nats

# Or manually
cd local/nats
./deploy.sh csc
./deploy.sh cpc-1
./deploy.sh cpc-2
```

### Deploy to Single Layer

```bash
# Deploy to CSC
helm install nats-event-bus ../../deploy/nats-event-bus \
  --namespace event-bus \
  --create-namespace \
  -f k8s/local-dev-values.yaml \
  -f k8s/csc/values.yaml \
  --kube-context kind-csc

# Deploy to CPC-1
helm install nats-event-bus ../../deploy/nats-event-bus \
  --namespace event-bus \
  --create-namespace \
  -f k8s/local-dev-values.yaml \
  -f k8s/cpc/values.yaml \
  -f k8s/cpc/cpc-1.yaml \
  --kube-context kind-cpc-1
```

## Configuration

This folder contains operator-facing configuration only. Internal NATS details (accounts, mappings, leaf nodes) are auto-generated by the `nats-event-bus` Helm chart based on `eventBus.clusterType`.

### Key Configuration

Set `eventBus.clusterType` to configure the cluster automatically:

```yaml
# CSC cluster
eventBus:
  clusterType: csc
  cpcIds: ["1", "2"]

# CPC cluster
eventBus:
  clusterType: cpc
  clusterId: "1"
  cscEndpoint: "nats://172.18.200.1:7422"
  crossLayer:
    cscExports: ["broadcast.>"]
    cscPrefixedExports: ["command.>"]
    cpcExports: ["sensor.>"]
```

Auth permissions are defined under `eventBus.auth.permissions`:

```yaml
eventBus:
  auth:
    permissions:
      oauth2:
        mqtt-client:
          azp: "mqtt-client"
          account: "CSC"  # or "CPC"
          permissions:
            pub: { allow: ["test.>"] }
            sub: { allow: ["test.>"] }
```

### MQTT Support

NATS provides native MQTT support:
- Listen on port 1883 (non-TLS)
- Support MQTT 3.1.1 protocol
- Handle QoS 0, 1 (QoS 2 optional)
- Maps MQTT topics to NATS subjects (dots replace slashes)

### JetStream Persistence

JetStream configuration:
- Memory-based storage
- Replicated across 3 nodes
- Auto-configured based on `clusterType`

### Federation via Leaf Nodes

CPC clusters connect to CSC as leaf nodes through the Envoy Gateway (port 7422). Topic filtering and transformation occurs in NATS account configuration, not at the Gateway. Leaf nodes and account configurations are auto-generated based on `eventBus.clusterType` and `eventBus.crossLayer` settings.

### Layer Isolation

Isolation is achieved through network and gateway-level enforcement:

- **Network Isolation**: Each layer runs in a separate Kubernetes cluster (CSC, CPC-1, CPC-2) with overlapping internal networks (10.244.0.0/16, 10.96.0.0/12)
- **Gateway Enforcement**: Inter-cluster communication goes through Envoy Gateway LoadBalancer services, enforcing layer boundaries
- **Topic Filtering**: NATS account configuration enforces which topics are allowed to pass between layers

This architecture ensures that clients in one layer cannot directly access the event bus in another layer.

## Testing

### Verify Deployment

```bash
make validate-nats
```

### Test MQTT Connectivity

```bash
kubectl port-forward -n event-bus-nats svc/nats 1883:1883 --context kind-csc
```

## Monitoring

### Prometheus Metrics

NATS exposes metrics on port 7777:
```bash
kubectl port-forward -n event-bus-nats svc/nats-metrics 7777:7777 --context kind-csc
curl http://localhost:7777/metrics
```

**Key Metrics:**
- `nats_server_in_msgs`: Incoming messages
- `nats_server_out_msgs`: Outgoing messages
- `nats_server_in_bytes`: Incoming bytes
- `nats_server_slow_consumers`: Slow consumer count
- `jetstream_stream_messages`: JetStream message count
- `jetstream_stream_bytes`: JetStream storage usage

### Grafana Dashboard

Import NATS dashboard ID 2279 in Grafana. See [NATS documentation](https://docs.nats.io/nats-server/configuration/monitoring) for details.

## Troubleshooting

### Pod Not Starting

```bash
kubectl get events -n event-bus-nats --context kind-csc
kubectl logs -n event-bus-nats <pod-name> --context kind-csc
```

### MQTT Connection Failed

Check MQTT is enabled in configuration. Verify Gateway TCPRoute is configured correctly.

### Federation Not Working

Check leaf node connections and Gateway configuration. Verify topic filtering at Gateway.

### High Memory Usage

Check JetStream stream usage and adjust retention policies as needed.
